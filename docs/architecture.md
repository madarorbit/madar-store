# معمارية مَدار

## الوضع الحالي

واجهة Next.js 16 باستخدام App Router وReact Server Components للعرض. تظل البيانات الثابتة الحالية في `src/data` إلى أن تُرحّل في migration مراقب. `src/config/site.ts` هو مصدر الهوية العامة، و`src/lib/commerce.ts` مصدر الثوابت غير السرية لمسار التحويل.

## معمارية الإنتاج المستهدفة

- **الويب:** Next.js App Router؛ Server Actions/Route Handlers للتحولات فقط، مع تحقق Zod على الخادم.
- **البيانات:** PostgreSQL، معاملات للطلب/الدفع، وفهارس موثقة في `database-schema.md`.
- **الهوية:** مزود جلسات موثوق، cookies `HttpOnly`, `Secure`, `SameSite=Lax`، وتفويض موجه بالدور داخل الخادم.
- **الملفات:** Object storage خاص؛ لا تحفظ الملفات في `public/`. Route Handler يتحقق من ملكية الطلب ثم يصدر URL موقّعاً قصير العمر.
- **البريد:** مزود عبر متغير بيئة فقط، مع outbox/queue للرسائل غير المتزامنة.

لا ينبغي اعتبار شاشة Checkout الحالية معالج دفع: تعطيل الإنشاء مقصود حتى ربط قاعدة بيانات وتخزين مخصصين، منعاً لفقد الطلبات أو إثباتات الدفع.
