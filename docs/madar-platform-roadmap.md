# خارطة تنفيذ منصة مَدار

## المرحلة المنفذة في هذا التغيير: التأسيس والأمان

- تثبيت الهوية والروابط الرسمية وSEO الأساسي.
- توثيق تدقيق المشروع وبنية الإطلاق.
- تعريف نموذج نطاق مستقل للطلبات والدفع والتحويل البنكي والتسليم.
- توثيق الحواجز الصلاحيات الخادمية المطلوبة للمالك ولوحة الإدارة.
- تجهيز متغيرات البيئة وSecurity Headers، مع توضيح أن واجهات الإدارة الحالية لا تصلح للإنتاج قبل المصادقة.
- توفير مسار Checkout واضح للتحويل البنكي وحالة مراجعة الدفع، دون الادعاء بمعالجة دفع أو رفع ملفات قبل إضافة التخزين وقاعدة البيانات.

## المرحلة التالية: الاستمرارية والمصادقة

1. إضافة PostgreSQL وORM، ثم migrations للكيانات الموثقة في `database-schema.md`.
2. إضافة مزود مصادقة بكلمات مرور مشفرة، تحقق بريد، استعادة كلمة المرور، rate limits وسجلات دخول.
3. تنفيذ seed آمن يمنح `SUPER_ADMIN` للبريد المطابق لـ`SUPER_ADMIN_EMAIL` بعد التحقق فقط.
4. ربط middleware/طبقة جلسات بحواجز الإدارة والخدمات الحساسة.

## المرحلة التالية: التجارة والتسليم

1. ترحيل المنتجات والخدمات والتصنيفات إلى القاعدة وإدارة CRUD محمية.
2. سلة دائمة، كوبونات، طلبات وحالات، ورفع إثبات محقق من النوع والحجم.
3. قبول/رفض الدفع كعملية ذرية مع AdminAuditLog.
4. مزود تخزين خاص وروابط تنزيل موقعة ومحدودة العمر والعدد.

## مرحلة الإطلاق

- تنفيذ اختبارات التكامل وE2E.
- مراجعة قانونية محلية للسياسات قبل النشر.
- إعداد البريد والتخزين والنسخ الاحتياطي والمراقبة وPostgreSQL الإنتاجية.
